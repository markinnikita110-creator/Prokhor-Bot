Сделай в моём Telegram-боте на Python надёжную систему автоматического исправления и обновления SQLite базы данных (“database self-healing system”).

🎯 Цель

База данных на телефоне (UserLAnd / Linux) должна автоматически приводиться в актуальное состояние при каждом запуске бота.
Никакие ошибки вида:

"no column named ..."
"no such table ..."
больше не должны ломать бота.
🧠 Требования
1. Обязательная функция при старте

Создай функцию:

def ensure_db_schema(conn: sqlite3.Connection):

Она должна выполняться ПЕРЕД запуском Telegram polling.

2. Функционал функции

Функция должна:

✔ Проверять все таблицы
через sqlite_master
создавать отсутствующие через CREATE TABLE IF NOT EXISTS
✔ Проверять все колонки
через PRAGMA table_info(table_name)
если колонки нет → добавлять через ALTER TABLE
3. Поведение (ОЧЕНЬ ВАЖНО)
НЕ удалять таблицы
НЕ удалять данные
НЕ пересоздавать базу
ТОЛЬКО добавлять недостающие элементы
4. Самовосстановление (CRITICAL FEATURE)

Если возникает ошибка при работе с БД:

sqlite3.OperationalError

бот должен:

поймать ошибку
автоматически повторно вызвать ensure_db_schema(conn)
повторить запрос ещё раз (retry 1 раз)
5. Архитектура

Создай файл:

db_guard.py

Внутри:

ensure_db_schema()
список всех таблиц и колонок (единый источник истины)
6. Структура схемы

Добавь централизованное описание схемы:

DB_SCHEMA = {
    "users": {
        "id": "INTEGER PRIMARY KEY",
        "tg_id": "INTEGER",
        "name": "TEXT",
        "language": "TEXT DEFAULT 'ru'"
    },
    "promo_codes": {
        "id": "INTEGER PRIMARY KEY",
        "code": "TEXT",
        "duration_days": "INTEGER"
    }
}
7. Логирование

Добавь понятные логи:

"DB OK"
"Created table: X"
"Added column: X to table Y"
"Schema already up to date"
8. Интеграция в main.py

В главном файле:

conn = sqlite3.connect("database.db")
ensure_db_schema(conn)

ДОЛЖНО выполняться ДО запуска бота.

9. Дополнительно (очень желательно)

Добавь:

защиту от двойного выполнения миграций (lock in-memory flag)
минимальный retry механизм для DB ошибок
🚀 Итог

Бот должен:

сам чинить базу
сам добавлять новые поля
никогда не падать из-за изменений схемы
работать стабильно на телефоне (UserLAnd)

Это критически важно для production-стабильности.
