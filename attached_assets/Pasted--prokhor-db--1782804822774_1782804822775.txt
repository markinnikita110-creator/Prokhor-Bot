# Задача: убрать prokhor.db из репозитория навсегда (включая историю)

## Контекст

В репозитории случайно закоммичен файл `prokhor.db` — это SQLite база с потенциальными персональными данными клиентов психологов (имена, заметки, telegram_id и т.д.). Файл нужно полностью убрать из репозитория, включая всю историю коммитов, и настроить так, чтобы он больше никогда туда не попадал.

## Шаги

### 1. Добавить в .gitignore

Добавь в файл `.gitignore` (в корне репозитория) следующие строки, если их там ещё нет:

```
# Local SQLite database — never commit, contains user PII
prokhor.db
*.db
*.db-journal
*.db-wal
*.db-shm
```

### 2. Удалить файл из истории репозитория полностью

Используй `git filter-repo` (предпочтительно) или `git filter-branch`, чтобы удалить `prokhor.db` из ВСЕЙ истории коммитов, а не только из текущего состояния:

```bash
# Установка filter-repo если не установлен
pip install git-filter-repo

# Удаление файла из всей истории
git filter-repo --path prokhor.db --invert-paths --force
```

Если `git filter-repo` недоступен в среде, используй альтернативу через `git filter-branch`:

```bash
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch prokhor.db" \
  --prune-empty --tag-name-filter cat -- --all

rm -rf .git/refs/original/
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

### 3. Принудительно запушить переписанную историю

```bash
git push origin --force --all
git push origin --force --tags
```

**Важно**: предупреди меня перед этим шагом, так как принудительный push перезаписывает историю на GitHub необратимо. После этого шага старые клоны репозитория (включая тот, что на моём телефоне) станут несовместимы с новой историей и потребуют переклонирования.

### 4. Сделать репозиторий приватным (если ещё не приватный)

Проверь настройки репозитория на GitHub — если репозиторий публичный, переключи его в приватный режим (Settings → Danger Zone → Change repository visibility → Make private). Это дополнительный уровень защиты на случай, если где-то в истории/кэшах GitHub/форках всё ещё остались следы файла.

### 5. Создать пустой prokhor.db не нужно

Не создавай заглушку файла — `database.py` уже содержит функцию `init_db()`, которая создаёт базу автоматически при первом запуске бота, если файла нет. Это правильное поведение — пусть так и остаётся.

### 6. Подтверди результат

После выполнения всех шагов проверь, что:
- `git log --all --full-history -- prokhor.db` ничего не находит (файла нет даже в истории).
- В текущем рабочем дереве `prokhor.db` отсутствует или, если есть локально, не отслеживается git (`git status` его не показывает).
- `.gitignore` содержит правило для `*.db`.

## Не трогай

- Саму логику `init_db()` и `migrate_db()` в `database.py` — она должна продолжать работать как раньше, просто файл базы больше не должен попадать в git.
