Задача: перенести домен "Домашние задания" в core-слой, по той же схеме, что заметки и клиенты. Работай строго поэтапно, останавливайся после каждого этапа и жди моего "продолжай". Веди работу в отдельной git-ветке (например, refactor/core-homework), не трогай main.

ЖЁСТКИЕ ПРАВИЛА (не меняются):
- Только перенос существующего кода, без изменения поведения.
- Найденные по пути баги — не чинить тихо, вынести в отчёт и спросить меня.
- Не добавлять новых функций.
- После каждого этапа — список изменённых файлов и что стало короче в старых местах.
- Использовать core/utils/time.py и core/db/base.py, уже созданные ранее — не дублировать.

ЭТАП 0 — Аудит (код не менять)
1. Найди в database.py все функции работы с домашними заданиями: назначение клиенту, чтение списка (по клиенту, по психологу), отметка выполнения/невыполнения, удаление.
2. Найди во всех handlers/*.py, где эти функции вызываются — список (файл, функция).
3. Проверь бизнес-правила: есть ли лимит по тарифу на количество активных домашних заданий, есть ли автоматическая привязка к клиенту/сессии, есть ли уведомление клиенту при назначении задания (и через какой механизм — напрямую send_message или через общую систему напоминаний/reminder_loop).
4. Проверь, участвуют ли домашние задания в клиентской стороне бота (то, что начинали проектировать в "Меню клиента" — раздел "Мои домашние задания") — если там уже есть код (даже частично), учти его в переносе, не оставляй отдельно необработанным.
5. Проверь, ссылаются ли задания на конкретную сессию (session_id) или существуют независимо.
6. Проверь прямые SQL-запросы к таблице заданий внутри хендлеров, минуя database.py.
7. Выведи результат текстом. ОСТАНОВИСЬ, дождись "продолжай".

ЭТАП 1 — Перенос
1. core/db/homework_repository.py — перенеси все функции работы с таблицей домашних заданий из database.py, используя core/db/base.py. Без изменения логики.
2. core/services/homework.py — вынеси найденные бизнес-правила (лимиты, уведомления при назначении), которые внутри вызывают core/db/homework_repository.py. Если правил нет — тонкая обёртка.
3. Удали перенесённые функции из database.py.
4. Поправь импорты во всех handlers/*.py (включая клиентскую сторону, если она уже написана) — на core/services/homework.py.

ЭТАП 2 — Отчёт и чек-лист ручной проверки
Список изменённых файлов и diff-сводка, плюс чек-лист:
- Назначить клиенту домашнее задание → появилось в его карточке
- Отметить задание выполненным (со стороны психолога, и со стороны клиента, если этот сценарий уже реализован) → статус обновился в обоих местах одинаково
- Список активных/выполненных заданий по клиенту → отображается верно
- Удалить задание (если такая функция есть) → пропало
- Если есть лимит по тарифу — проверить, что срабатывает как раньше
- Уведомление клиенту о новом задании (если реализовано) — приходит как раньше
- Полное удаление практики психолога → домашние задания клиента реально удаляются вместе с остальным

Не переходи к переносу других доменов в рамках этой задачи.
